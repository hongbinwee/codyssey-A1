# Codyssey A1 미션 한 페이지 요약

## 1. 미션 핵심 요약
이번 미션의 본질은 `파이썬으로 동작하는 콘솔 프로그램`을 만들고, 그 과정을 `Git/GitHub 이력`으로 남기는 것입니다. 단순히 문법을 외우는 과제가 아니라, 아이디어를 기능으로 구현하고 변경 기록까지 관리하는 흐름을 직접 경험하는 것이 목표입니다.

## 2. 필수 산출물
- Python 콘솔 프로그램 1개
- GitHub 저장소 1개
- README.md
- 개발 환경 설정 스크린샷
- 프로그램 실행 결과 스크린샷
- `git log --oneline --graph` 스크린샷

## 3. 선택 또는 보너스 과제
- JSON 파일 저장/불러오기
- 카테고리별 Markdown 내보내기
- 프롬프트 수정/삭제
- 상세 보기 조회수 기록
- 조회수 기준 정렬

## 4. 기능 요구사항 체크리스트
- 메뉴 기반 콘솔 프로그램이어야 함
- 프롬프트 추가, 목록, 카테고리 조회, 검색, 상세 보기, 즐겨찾기 기능 필요
- 프로그램 시작 시 기본 프롬프트 3개 이상 포함
- 데이터는 리스트와 딕셔너리 사용
- 기능별 함수 분리
- 잘못된 입력 처리 필요
- 프로그램 종료 전까지 데이터 유지

## 5. 과제 목표 연결
이 과제를 끝내면 아래를 스스로 설명할 수 있어야 합니다.
- Python 파일 생성 및 실행
- Python/Git 버전 확인
- 기본 문법과 함수 분리
- Git의 역할
- `init`, `add`, `commit`, `push`, `pull`, `checkout`, `clone`, `merge`
- 브랜치 생성 및 병합
- GitHub 업로드 및 관리

## 6. 추천 진행 순서
1. 개발 환경 확인
2. Git 저장소 초기화
3. 기본 파일 생성
4. 메뉴 기능부터 구현
5. 프롬프트 데이터 구조 작성
6. 기능별 함수 분리
7. README 작성
8. 기능 단위 커밋
9. 브랜치 생성/병합 기록 남기기
10. 실행 및 제출 자료 캡처

## 7. 제출 전 검증 체크리스트
- 메뉴가 정상 출력되는가
- 각 기능 후 다시 메뉴로 돌아가는가
- 빈 입력과 잘못된 번호를 처리하는가
- 기본 프롬프트 3개 이상 있는가
- 즐겨찾기 추가/해제가 되는가
- README에 설명, 실행법, 기능 목록, 카테고리가 있는가
- GitHub에 코드가 올라갔는가
- 의미 있는 커밋 10개 이상 있는가
- 브랜치 생성 및 병합 기록이 있는가

## 8. 확인 질문
- 기본 프롬프트 3개는 이전 미션 내용을 반영했는가
- 카테고리 선택 방식이 사용자에게 충분히 쉬운가
- 함수 분리가 명확한가
- 캡처해야 할 화면을 모두 준비했는가

## 9. 개념 이해용 추가 학습 프롬프트
- `파이썬 리스트와 딕셔너리를 초보자 예시로 설명해줘`
- `while문으로 메뉴 프로그램 만드는 방법을 설명해줘`
- `git add, commit, push 차이를 쉬운 비유로 설명해줘`
- `브랜치와 merge가 왜 필요한지 초보자에게 설명해줘`
